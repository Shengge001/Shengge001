 题目描述（中文理解）

 

 第一题：Break repeating-key XOR（Cryptopals Set 1 Challenge 6）

这道题给出一个经过 Base64 编码的密文，它是由一个未知长度的密钥进行重复密钥 XOR（类似维吉尼亚密码）加密得到的。我们需要在不知道密钥的情况下恢复出明文。核心思想是：由于密钥重复使用，可以将密文按密钥长度分成若干列，每一列实际上是用同一个单字节密钥进行 XOR 加密。因此，先猜测密钥长度，再分别破解每个密钥字节，最后用完整密钥解密。

 第二题：Cracking SHA1-Hashed Passwords（MTC3 Level 2）

这道题给出一个管理员密码的 SHA1 哈希值，同时提供了键盘磨损的照片。通过分析键盘上哪些键被频繁使用（磨损严重），可以推断出密码可能包含的字符集合。然后利用这些字符作为候选集，进行字典攻击或暴力枚举，找到与给定哈希匹配的密码。不需要从头猜测所有可能，而是基于物理线索缩小搜索空间。

一、破解重复密钥异或加密（Break repeating-key XOR）

1. 背景

 

重复密钥 XOR 加密是古典密码维吉尼亚密码的二进制版本。当密钥长度小于明文长度时，密钥会循环使用。这种加密在不知道密钥的情况下，可以利用统计特性破解。

 

2. 原理

 

・ 汉明距离：两个等长字符串之间不同比特的个数。对于随机字节串，归一化汉明距离约为 4；对于自然语言文本，归一化汉明距离较小。

・ 密钥长度猜测：取不同密钥长度 KEYSIZE，将密文分成多个 KEYSIZE 大小的块，计算块间平均归一化汉明距离，最小值对应的 KEYSIZE 即为真实密钥长度。

・ 单字节 XOR 破解：将密文按密钥长度转置后，每一列独立地用频率分析法破解：尝试 0~255 作为密钥字节，解密后统计英文字母和空格的个数，得分最高的即为正确密钥字节。

 

3. 必要的公式

 

・ 汉明距离：

  \text{Hamming}(a,b) = \sum_{i=0}^{n-1} \text{popcount}(a_i \oplus b_i)

  其中 popcount 统计一个字节中 1 的个数。

・ 归一化距离：

  \text{norm\_dist} = \frac{\text{平均汉明距离}}{\text{KEYSIZE}}

 

4. 实现步骤

 

1. 读取 6.txt 文件，去除换行符，进行 Base64 解码得到密文字节串。

2. 定义汉明距离计算函数。

3. 对于 KEYSIZE 从 2 到 40：

   ・ 取前 4 个 KEYSIZE 长度的块。

   ・ 计算所有块两两之间的汉明距离，求和后除以块对数，再除以 KEYSIZE 得到归一化距离。

   ・ 记录最小归一化距离对应的 KEYSIZE。

4. 将密文按 KEYSIZE 分成若干列（转置）：第 i 列由密文中所有索引 i, i+KEYSIZE, i+2KEYSIZE,… 的字节组成。

5. 对每一列调用单字节 XOR 破解函数：

   ・ 尝试所有 0~255 作为密钥字节。

   ・ 解密该列，统计结果中属于 etaoinshrdlu ETAOINSHRDLU ,.?! 的字符数量。

   ・ 取最高分对应的密钥字节。

6. 将所有列的密钥字节按顺序拼接成完整密钥。

7. 用该密钥对原始密文逐字节 XOR 解密，得到明文。

8. 输出解密结果的前 500 字符和完整密钥。

 

5. 代码

 

```python

import base64

import time

 

def hamming_distance(b1, b2):

    return sum(bin(x ^ y).count('1') for x, y in zip(b1, b2))

 

def guess_keysize(ciphertext, max_keysize=40):

    best_keysize = None

    best_norm = float('inf')

    for keysize in range(2, max_keysize + 1):

        blocks = [ciphertext[i*keysize:(i+1)*keysize] for i in range(4)]

        total_dist = 0

        pairs = 0

        for i in range(4):

            for j in range(i+1, 4):

                total_dist += hamming_distance(blocks[i], blocks[j])

                pairs += 1

        norm_dist = total_dist / pairs / keysize

        if norm_dist < best_norm:

            best_norm = norm_dist

            best_keysize = keysize

    return best_keysize

 

def break_single_byte_xor(block):

    common = b"etaoinshrdlu ETAOINSHRDLU ,.?!"

    best_score = 0

    best_key = None

    for key in range(256):

        plain = bytes([b ^ key for b in block])

        score = sum(1 for c in plain if c in common)

        if score > best_score:

            best_score = score

            best_key = key

    return best_key

 

def break_repeating_key_xor(ciphertext):

    keysize = guess_keysize(ciphertext)

    print(f"[+] 猜测密钥长度: {keysize}")

    blocks = [ciphertext[i::keysize] for i in range(keysize)]

    key_bytes = []

    for i, blk in enumerate(blocks):

        key_byte = break_single_byte_xor(blk)

        key_bytes.append(key_byte)

        print(f"    第 {i+1:2d} 列 密钥字节 = {key_byte:02x} ('{chr(key_byte)}')")

    key = bytes(key_bytes)

    print(f"\n[+] 完整密钥: {key.decode('ascii')}")

    plaintext = bytes([ciphertext[i] ^ key[i % keysize] for i in range(len(ciphertext))])

    return plaintext

 

if __name__ == "__main__":

    start = time.time()

    with open("6.txt", "r") as f:

        b64data = f.read().replace('\n', '')

    ciphertext = base64.b64decode(b64data)

    print(f"[*] 密文长度: {len(ciphertext)} 字节")

    result = break_repeating_key_xor(ciphertext)

    print("\n[+] 解密结果（前500字符）:\n")

    print(result[:500].decode('ascii', errors='replace'))

    with open("decrypted.txt", "wb") as f:

        f.write(result)

    print(f"\n[+] 运行时间: {time.time()-start:.2f} 秒")

```

二、破解SHA1哈希密码（Cracking SHA1-Hashed Passwords）

1. 背景

 

在密码存储中，通常不保存明文密码，而是保存其哈希值（如 SHA1）。当用户登录时，系统计算输入密码的哈希并与存储值比较。如果攻击者获得了哈希值，需要通过暴力或字典攻击找到原像。本题额外提供了键盘磨损痕迹，大大减少了可能的密码组合。

 

2. 原理

 

・ SHA1 哈希算法：将任意长度的输入转换为 160 位（40 个十六进制字符）的固定长度输出，具有单向性和抗碰撞性。

・ 键盘磨损分析：经常按下的键会失去表面的字符印刷或出现油亮痕迹。左右 Shift 磨损说明密码中包含大写字母或符号；数字键和符号键磨损说明对应字符出现在密码中。

・ 搜索策略：根据磨损键生成候选字符集，然后生成所有可能的字符串（限定长度范围），计算每个字符串的 SHA1 并与目标哈希比较，匹配即成功。

 

3. 步骤

 

1. 分析键盘磨损图

   ・ 磨损键：左 Shift、右 Shift、回车、@、q、w、i、n、5、%、8、(、[、]、0、=、~、*、+，以及右侧数字小键盘的 2、4、6、8（方向键）。

   ・ 可输入字符（忽略功能键）：

          小写字母：q, w, i, n

          大写字母：Q, W, I, N（因为有 Shift）

          数字：5, 8, 0, 2, 4, 6

          符号：%, (, ), [, ], =, +, @, ~, *

2. 确定密码长度范围

      通常密码长度在 5~8 之间（根据键盘磨损的键位数量估计）。

3. 编写破解程序（代码不在此展示，但描述逻辑）

   ・ 读入目标哈希字符串。

   ・ 定义候选字符列表。

   ・ 使用多重循环或 itertools.product 生成所有长度从 5 到 8 的组合。

   ・ 对每个组合计算 SHA1 并比较。

   ・ 匹配成功则输出密码并终止。

4. 主要实现代码（关键部分）

import hashlib

import itertools

import datetime

 

start_time = datetime.datetime.now()

 

target_hash = "67ae1a64661ac8b4494666f58c4822408dd0a3e4"

base_chars = "QqWw%58(=0Ii*+nN"

choice_pairs = [['Q', 'q'], ['W', 'w'], ['%', '5'], ['8', '('],

                ['=', '0'], ['I', 'i'], ['*', '+'], ['n', 'N']]

 

# 存储每一位的选择

selected = [0] * 8

password_found = ""

 

# 8层循环枚举每一位的大小写/符号变体

for idx0 in range(2):

    selected[0] = choice_pairs[0][idx0]

    for idx1 in range(2):

        selected[1] = choice_pairs[1][idx1]

        for idx2 in range(2):

            selected[2] = choice_pairs[2][idx2]

            for idx3 in range(2):

                selected[3] = choice_pairs[3][idx3]

                for idx4 in range(2):

                    selected[4] = choice_pairs[4][idx4]

                    for idx5 in range(2):

                        selected[5] = choice_pairs[5][idx5]

                        for idx6 in range(2):

                            selected[6] = choice_pairs[6][idx6]

                            for idx7 in range(2):

                                selected[7] = choice_pairs[7][idx7]

                                base_password = "".join(selected)

                                # 对当前8个字符的组合进行全排列

                                for perm in itertools.permutations(base_password, 8):

                                    guess = "".join(perm)

                                    hashed = hashlib.sha1(guess.encode("utf-8")).hexdigest()

                                    if hashed == target_hash:

                                        password_found = guess

                                        end_time = datetime.datetime.now()

                                        print(f"密码是：{guess}")

                                        print(f"运行时间是：{(end_time - start_time).seconds}秒")

总结（完成心得与其它，主要自己碰到的问题和解决问题的方法）

在第一题（Break repeating-key XOR）中，主要遇到两个问题：一是文件读取失败，原因是 Windows 隐藏了文件扩展名，实际文件名为 6.txt.txt，通过修改代码中的文件名或重命名文件解决；二是解密结果出现乱码，通过增加常见字符集（包含空格和标点）并取多个块计算汉明距离平均值，提高了密钥长度猜测的准确性。最终成功猜测密钥长度为 29，密钥为 Terminator X: Bring the noise，解密出的明文以 "I'm back and I'm ringin' the bell..." 开头。

 

在第二题（Cracking SHA1-Hashed Passwords）中，核心挑战是根据键盘磨损痕迹缩小密码搜索空间。初始根据常见规律猜测密码为 QWin%5，但计算其 SHA1 后发现与题目给出的哈希（67ae1a64661ac8b4494666f58c4822408dd0a3e4）不匹配。随后重新分析磨损键，整理出完整的候选字符集（包括大小写字母、数字及 % ( ) [ ] = + @ ~ * 等符号）。由于候选字符较多（约 24 个），暴力枚举时间较长，因此采用优先尝试常见键盘顺序组合的策略，并结合字典攻击。最终通过程序枚举找到正确密码。解决的关键是准确识别 Shift 磨损所代表的大写和符号可能，以及合理限定密码长度范围（5~8 位）。
